# 自我进化系统规范 (Self-Evolution System)

```typescript
/**
 * 自我进化系统规范
 * @description 定义 Agent 自主感知、决策、学习、优化的行为准则
 */
export interface SelfEvolutionSystem {
  /**
   * 跨会话记忆
   */
  crossSessionMemory: {
    /** 记忆存储位置 */
    storagePath: string;
    /** 记忆类型 */
    memoryTypes: ('episodic' | 'semantic' | 'procedural')[];
    /** 写入时机 */
    writeTriggers: string[];
    /** 读取时机 */
    readTriggers: string[];
  };

  /**
   * 项目上下文感知
   */
  contextAwareness: {
    /** 启动检查清单 */
    startupChecklist: string[];
    /** 上下文刷新触发器 */
    refreshTriggers: string[];
    /** 必须了解的项目信息 */
    requiredKnowledge: string[];
  };

  /**
   * 自主搜索能力
   */
  autonomousSearch: {
    /** 触发搜索的条件 */
    searchTriggers: string[];
    /** 信息源优先级 */
    sourcePriority: string[];
    /** 搜索结果处理 */
    resultProcessing: string[];
  };

  /**
   * 自我反思机制
   */
  selfReflection: {
    /** 反思触发器 */
    reflectionTriggers: string[];
    /** 反思内容 */
    reflectionTopics: string[];
    /** 进化行动 */
    evolutionActions: string[];
  };
}
```

## 1. 跨会话记忆系统

### 1.1 记忆存储结构
```
.trae/memory/
├── SUMMARY.md          # 记忆摘要（启动时读取）
├── sessions/           # 会话记忆（发生了什么）
│   └── YYYY-MM-DD.md   # 按日期记录
├── archive/            # 归档记忆（30天+）
│   └── YYYY-MM/        # 按月份归档
│       └── YYYY-MM-DD.md
├── capabilities/       # 能力记忆（会做什么）
│   └── index.md        # 能力清单
├── decisions/          # 决策记忆（为什么这么做）
│   └── index.md        # 决策日志
└── patterns/           # 模式记忆（规律总结）
    └── index.md        # 行为模式
```

### 1.2 记忆生命周期管理

| 阶段 | 时间范围 | 处理方式 | 说明 |
|:---|:---|:---|:---|
| **热记忆** | 0-7 天 | 完整保留 | 近期记忆，最常使用 |
| **温记忆** | 7-30 天 | 压缩摘要 | 压缩为摘要，减少体积 |
| **冷记忆** | 30 天+ | 归档存储 | 移至 archive/，按需检索 |

**触发时机**：每周执行一次 `memory_lifecycle.py`

### 1.3 记忆读取策略（轻量级）
- **启动时**：只读取 `SUMMARY.md`（约 30 行）
- **按需深入**：任务需要时才读取详情文件
- **原则**：最小化 token 消耗，最大化信息密度

### 1.4 记忆写入时机
- **会话结束时**：记录本次会话的关键活动、决策、成果
- **完成任务后**：记录任务过程、遇到的问题、解决方案
- **写入后**：自动更新 `SUMMARY.md`

### 1.5 记忆与知识引擎的分工与配合

#### 分工边界

| 维度 | 记忆系统 | 知识引擎 |
|:---|:---|:---|
| **记录什么** | 发生了什么（叙事） | 怎么解决（方法论） |
| **内容形式** | 时间线 + 活动列表 | 规则 + 模板 + 代码示例 |
| **调用方式** | 自动记录 | 按需记录（有价值时） |
| **生命周期** | 会衰减（热→温→冷） | 永久保留 |
| **用途** | 了解上下文 | 解决问题 |

#### 触发时机对比

| 场景 | 记忆系统 | 知识引擎 |
|:---|:---|:---|
| 完成任务 | ✅ 记录活动 | ❌ 不记录（除非有价值） |
| 解决 Bug | ✅ 记录修复过程 | ✅ 记录解决方案（有复用价值） |
| 做出决策 | ✅ 记录决策理由 | ✅ 记录决策模式（有参考价值） |
| 学习新知识 | ✅ 记录学习内容 | ✅ 记录知识点（有查阅价值） |
| 日常对话 | ✅ 记录对话摘要 | ❌ 不记录 |

#### 配合机制

```
┌─────────────────────────────────────────────────────────────┐
│                    交互发生                                  │
└────────────────────────┬────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────────┐
│  第一步：记忆系统记录（自动）                                │
│  - 记录"发生了什么"                                         │
│  - 更新 SUMMARY.md                                          │
└────────────────────────┬────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────────┐
│  第二步：判断价值（自动）                                    │
│  - 是否有复用价值？                                         │
│  - 是否有参考价值？                                         │
│  - 是否有查阅价值？                                         │
└────────────┬────────────────────────────────────────────────┘
             ↓                               ↓
        有价值 ✅                        无价值 ❌
             ↓                               ↓
┌────────────────────────┐         ┌────────────────────────┐
│ 第三步：触发知识引擎    │         │ 结束                   │
│ - 调用 knowledge-gardener│         │                        │
│ - 按规范格式记录        │         │                        │
│ - 存入 Inbox            │         │                        │
└────────────────────────┘         └────────────────────────┘
```

#### 价值判断标准

| 价值类型 | 判断标准 | 示例 |
|:---|:---|:---|
| **复用价值** | 同类问题可能再次遇到 | Bug 修复方案、性能优化技巧 |
| **参考价值** | 决策逻辑可参考 | 技术选型理由、架构决策 |
| **查阅价值** | 知识点需要查阅 | API 用法、配置说明 |

## 2. 项目上下文感知

### 2.1 启动检查清单（轻量级）

| 序号 | 检查项 | 动作 |
|:---:|:---|:---|
| 1 | 读取记忆摘要 | `Read .trae/memory/SUMMARY.md` |
| 2 | 扫描项目变更 | `git status` + `git log -3 --oneline` |
| 3 | 向用户报告 | 展示上下文摘要（1-2句话） |

> **按需深入**: 只有当任务需要时，才读取详情文件

### 2.2 必须了解的项目信息

| 信息类型 | 获取方式 | 更新频率 |
|:---|:---|:---|
| 技术栈 | `Read .trae/rules/core/tech-stack.ts.md` | 按需 |
| 代码规范 | `Read .trae/rules/core/coding-style.ts.md` | 按需 |
| 工作流 | `Read .trae/rules/core/workflows.ts.md` | 按需 |
| 最近变更 | `git log -10 --oneline` | 每次启动 |
| 未完成事项 | `Read .trae/memory/sessions/最新文件` | 每次启动 |

### 2.3 上下文刷新触发器
- 用户说"继续上次的工作"
- 用户提到之前的某个任务
- 发现代码与记忆不一致
- 项目结构发生变化

## 3. 自主搜索能力

### 3.1 触发搜索的条件

| 条件 | 描述 | 搜索策略 |
|:---|:---|:---|
| **未知 API** | 遇到不熟悉的库/框架 API | 官方文档优先 |
| **报错信息** | 遇到错误/异常 | 错误信息 + 解决方案 |
| **技术选型** | 需要选择技术方案 | 对比分析 + 最佳实践 |
| **性能问题** | 性能瓶颈/优化需求 | 优化方案 + 基准测试 |
| **新领域** | 进入不熟悉的领域 | 入门教程 + 核心概念 |

### 3.2 信息源优先级
1. **官方文档** - 最权威
2. **GitHub Issues/Discussions** - 实际问题解决
3. **权威技术博客** - 深度分析
4. **Stack Overflow** - 常见问题
5. **技术社区** - 经验分享

### 3.3 搜索结果处理流程
```
搜索 → 筛选(相关性+权威性) → 验证(可行性) → 整合(最佳方案) → 记录(沉淀知识)
```

### 3.4 搜索决策报告
当完成重要搜索后，应向用户报告：
```markdown
## 搜索决策报告

### 问题
[描述遇到的问题]

### 搜索结果
- 方案A：[来源] + [优缺点]
- 方案B：[来源] + [优缺点]

### 推荐方案
[选择的方案] + [理由]

### 参考
- [链接1]
- [链接2]
```

## 4. 自我反思机制

### 4.1 反思触发器
- 完成复杂任务后
- 遇到重复性问题
- 用户反馈不满意
- 发现知识盲区
- 定期（每5次会话）

### 4.2 反思内容

| 反思维度 | 问题模板 |
|:---|:---|
| **效率** | "这次任务有没有更高效的方法？" |
| **质量** | "解决方案是否足够优雅？有没有遗漏？" |
| **学习** | "学到了什么新知识？值得记录吗？" |
| **盲区** | "有没有暴露知识盲区？如何补齐？" |
| **进化** | "系统能力是否需要增强？如何增强？" |

### 4.3 进化行动

| 触发条件 | 进化行动 |
|:---|:---|
| 发现知识盲区 | 创建学习任务 → 搜索学习 → 记录到知识库 |
| 发现重复模式 | 提炼规则 → 写入规范 → 创建自动化脚本 |
| 发现效率瓶颈 | 分析原因 → 优化流程 → 更新工作流 |
| 发现能力缺失 | 设计能力 → 实现机制 → 记录到能力清单 |

## 5. 行为准则

### 5.1 禁止事项
- ❌ 禁止在未了解项目上下文的情况下开始工作
- ❌ 禁止在遇到未知问题时凭猜测行动
- ❌ 禁止忽略搜索结果直接使用自己的假设
- ❌ 禁止完成任务后不更新记忆系统

### 5.2 强制事项
- ✅ 每次会话开始必须执行启动检查清单
- ✅ 遇到未知问题必须先搜索再行动
- ✅ 完成重要任务后必须更新记忆
- ✅ 做出重要决策后必须记录理由
- ✅ 发现改进机会必须主动提出

## 7. 与现有系统集成

### 7.1 与知识引擎集成
- 记忆系统作为知识引擎的扩展层
- 会话记忆与 Inbox 互补
- 能力记忆与规则库联动

### 7.2 与技能系统集成
- 启动检查清单可封装为技能
- 记忆更新可集成到现有技能
- 反思机制可触发技能优化
