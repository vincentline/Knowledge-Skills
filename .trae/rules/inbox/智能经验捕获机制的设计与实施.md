# 智能经验捕获机制的设计与实施

## Key Points

### 核心设计
- **分类处理机制**：
  - 开发任务：整合进 coder 技能，作为任务完成后的强制检查步骤
  - 非开发任务：Agent 主动识别，在合适时机提醒用户
- **三大判断维度**：
  - 稀缺性：Agent 首次遇到的问题、项目独创的技术方案
  - 决策价值：技术选型、架构决策、关键配置
  - 问题解决价值：Bug 修复、性能优化、坑点规避
- **交互设计**：AskUserQuestion 直接列出经验摘要，用户确认后调用 knowledge-gardener 记录

### 独创技术方案判断特征
- 官方文档/社区没有现成解决方案
- 需要自行设计实现方案
- 具有项目特色的技术处理方式
- 可能涉及对现有库/框架的扩展或改造

## Use Cases

1. **开发任务完成后**：coder 技能自动触发经验检查，回顾任务过程，判断是否有值得总结的经验
2. **非开发对话中**：Agent 持续关注对话内容，在合适时机询问用户是否记录经验

## Examples

### AskUserQuestion 设计示例
```
问题：发现以下经验值得记录，是否保存到 Inbox？
- 改用 xxx 技术栈的决策理由
- SVGA 音频暂停与进度调节的实现方案

选项：
1. "是，记录下来"
2. "否，不需要"
```
