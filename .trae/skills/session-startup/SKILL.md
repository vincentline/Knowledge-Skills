# Session Startup Skill

## 功能描述
会话启动时强制执行记忆激活和上下文感知，确保 Agent 具备完整的项目上下文后再开始工作。

## 触发条件
**每次会话第一条消息时必须执行**

## 执行流程

### Phase 1: 记忆激活
```
1. 读取最近会话记忆
   - 文件: .trae/memory/sessions/YYYY-MM-DD.md (最新的)
   - 目的: 知道之前发生了什么

2. 读取能力清单
   - 文件: .trae/memory/capabilities/index.md
   - 目的: 知道自己会做什么

3. 读取决策日志
   - 文件: .trae/memory/decisions/index.md
   - 目的: 知道为什么这么做
```

### Phase 2: 项目感知
```
4. 扫描项目变更
   - 命令: git status
   - 命令: git log -5 --oneline
   - 目的: 了解最新状态

5. 检查 Inbox
   - 文件: .trae/rules/inbox/index.md
   - 目的: 查看未归档经验
```

### Phase 3: 状态报告
```
6. 向用户报告上下文摘要
   - 最近做了什么
   - 有什么待办事项
   - 项目有什么变更
```

## 输出格式

```markdown
## 🧠 记忆激活完成

### 最近活动
- [日期] 活动1
- [日期] 活动2

### 待办事项
- [ ] 待办1
- [ ] 待办2

### 项目变更
- [变更摘要]

### 准备就绪
我已了解项目上下文，请告诉我需要做什么。
```

## 强制规则

1. **禁止跳过**：任何会话开始都必须执行
2. **禁止简化**：必须完成所有5个检查项
3. **必须报告**：必须向用户展示上下文摘要

## 与其他技能的关系

```
session-startup (启动)
    │
    ├──▶ coder (开发任务)
    ├──▶ knowledge-gardener (经验捕获)
    └──▶ 其他技能...
```

## 错误处理

| 情况 | 处理方式 |
|:---|:---|
| 记忆文件不存在 | 创建新文件，报告"首次会话" |
| git 命令失败 | 跳过变更扫描，报告"无法获取变更" |
| 文件读取失败 | 报告错误，继续执行其他检查 |

## 示例对话

**用户**: 帮我实现一个新功能

**Agent (执行 session-startup)**:
```
## 🧠 记忆激活完成

### 最近活动 (2026-03-14)
- 设计了自我进化系统
- 创建了跨会话记忆机制
- 你说明天要考验我

### 待办事项
- [ ] 测试跨会话记忆效果

### 项目变更
- 新增 .trae/memory/ 目录
- 新增 self-evolution.ts.md

### 准备就绪
我已了解项目上下文，请告诉我需要实现什么功能。
```
