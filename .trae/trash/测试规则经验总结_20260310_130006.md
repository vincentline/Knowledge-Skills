# 测试规则经验总结

## Key Points
- **工具选择策略**：优先使用 IDE 内置测试工具，无内置工具时使用 webapp-testing 技能
- **浏览器模式要求**：禁止无头浏览器模式，使用 browser-use 时必须添加 `--browser real` 参数
- **测试准备工作**：测试前查找项目内测试模板、测试用例、测试流程，参考现有测试资源
- **测试文件管理**：需要 svga、图片、视频等测试文件时，在项目根目录的 `test_files` 中查找
- **测试后处理流程**：删除冗余调试代码，优化测试模板，执行 GitHub 提交，提醒整理经验文档
- **标准执行流程**：检查内置测试工具 → 查找现有测试资源 → 选择测试工具 → 执行测试 → 后处理

## Use Cases
- **新项目测试**：为新项目建立测试流程和规范
- **测试工具选择**：在不同开发环境中选择合适的测试工具
- **测试资源管理**：规范测试文件的存放和使用
- **测试问题排查**：解决测试过程中遇到的常见问题
- **测试流程优化**：根据实际情况优化测试流程
