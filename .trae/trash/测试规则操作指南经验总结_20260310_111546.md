# 测试规则操作指南经验总结

## Key Points
- **测试流程标准化**：建立了完整的测试操作流程，包括前置条件、操作步骤、验证方法等
- **工具选择策略**：明确了测试工具的选择优先级，优先使用IDE内置测试工具，备选使用webapp-testing技能
- **资源管理规范**：规定了测试文件的存放位置和支持的文件类型
- **测试后处理**：包含了测试完成后的清理和优化步骤
- **常见问题解决方案**：提供了针对常见测试问题的解决方法

## Use Cases
1. **新项目测试**：为新项目建立测试流程和规范
2. **测试工具选择**：在不同场景下选择合适的测试工具
3. **测试资源管理**：规范测试文件的存放和管理
4. **测试问题排查**：解决测试过程中遇到的常见问题
5. **测试流程优化**：根据实际情况优化测试流程
