# 测试规则

## Key Points
- **工具选择**: 优先使用 IDE 内置测试工具，无内置工具时使用 webapp-testing 技能
- **浏览器模式**: 禁止无头浏览器模式，使用 browser-use 时添加 `--browser real` 参数
- **测试准备**: 测试前查找项目内测试模板、测试用例、测试流程，参考现有测试资源
- **测试文件**: 需要 svga、图片、视频等测试文件在项目根目录的 `test_files` 中找
- **测试后处理**: 删除冗余调试代码，新建或优化测试模板，执行 GitHub 提交，提醒整理经验文档
- **执行流程**: 检查内置测试工具 → 查找现有测试资源 → 选择测试工具 → 执行测试 → 后处理

## Use Cases
- 项目测试流程规范
- 测试工具选择指导
- 测试资源管理
- 测试后处理标准化

## Examples
- 优先使用 IDE 内置测试工具进行单元测试
- 无内置工具时，使用 webapp-testing 技能进行端到端测试
- 测试时使用真实浏览器模式，添加 `--browser real` 参数
- 测试文件从 `test_files` 目录获取，避免重复创建
- 测试完成后清理调试代码，更新测试模板，提交代码并整理经验
