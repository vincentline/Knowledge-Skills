# 模式记忆

> 记录从多次交互中提炼的行为模式、问题解决模式、效率模式

## 行为模式

### ⚠️ 用户问题响应模式（强制）
```
收到用户消息 → 读用户画像 → 读会话记忆 → 理解上下文 → 再回应
```
> **失败案例**: 用户问"敖隐有皮肤吗"，我没有先读用户画像，直接搜索回答了。实际上用户画像已记录敖隐是无皮肤英雄！

### 开发任务模式
```
识别领域 → 查阅规则 → 查看经验 → 编码实现 → 测试验证 → 记录经验
```

### 问题解决模式
```
遇到问题 → 搜索方案 → 评估选项 → 选择最优 → 实施验证 → 记录经验
```

### 知识管理模式
```
发现经验 → 结构化重写 → 归档到规则库 → 诊断检查
```

## 问题解决模式

### 常见问题类型及解决策略

| 问题类型 | 典型特征 | 解决策略 |
|:---|:---|:---|
| API 不熟悉 | 不知道某个库怎么用 | 查官方文档 → 搜示例代码 |
| 报错信息 | 有明确的错误提示 | 搜错误信息 → 查 Issues |
| 性能问题 | 运行慢、卡顿 | 分析瓶颈 → 搜优化方案 |
| 兼容性问题 | 某环境不工作 | 搜兼容性方案 → 查版本要求 |
| 设计决策 | 多个方案可选 | 列出选项 → 权衡利弊 → 推荐最优 |

## 效率模式

### 高效行为
- ✅ 批量读取文件（一次请求多个文件）
- ✅ 并行执行独立任务
- ✅ 先搜索再行动（避免猜测）
- ✅ 模仿现有代码风格

### 低效行为（应避免）
- ❌ 逐个读取文件
- ❌ 凭猜测写代码
- ❌ 忽略现有模式重新发明轮子
- ❌ 不搜索直接问用户

## 进化模式

### 能力增强触发器
- 重复做同一件事 3 次 → 考虑自动化
- 遇到知识盲区 → 主动学习并记录
- 用户反馈不满意 → 反思改进
- 发现更好的方法 → 更新规则

### 知识沉淀触发器
- 解决了新类型问题 → 记录解决模式
- 做出了重要决策 → 记录决策理由
- 学到了新知识 → 记录到能力清单
- 发现了规律 → 记录到模式记忆
